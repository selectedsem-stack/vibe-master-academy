מה זה Python בכלל?

Python היא שפת תכנות. כשאתה כותב קוד Python, אתה כותב הוראות למחשב בשפה שיחסית קלה לבני אדם לקרוא. קובץ Python (סקריפט) הוא קובץ טקסט שמסתיים ב-.py ושמריצים אותו מה-Terminal.

דוגמה — איך נראה קוד Python:

hello.py 
print("Hello, Shai!")

זאת שורה אחת — היא מבקשת מהמחשב להדפיס את המילים האלה למסך. עוד מעט נכתוב אותה בעצמנו ב-VS Code ונריץ אותה ב-Terminal.

Terminal — שלוש פקודות שתשתמש בהן כל הזמן

ה-Terminal הוא חלון שבו אתה מקליד פקודות ישירות למחשב. שלוש הפקודות הבסיסיות:

פקודהמה היא עושה
pwdמציגה את התיקייה הנוכחית
lsמציגה את הקבצים בתיקייה
cd <תיקייה>נכנסת לתיקייה. cd ~ חוזר לתיקיית הבית.
אם פקודה לא עובדת

קרא את הודעת השגיאה. command not found = הכלי לא מותקן. Permission denied = חסרות הרשאות (נסה sudo לפני הפקודה — יבקש סיסמה).

דוגמה 1 — סקריפט שמדבר איתך

30 דקות
מאיפה אנחנו

Python ו-VS Code מותקנים. הרצת hello.py. עכשיו נראה סקריפט קצת יותר מעניין — כזה ששואל אותך שאלה, מקבל ממך תשובה, ומשתמש בה. אתה לא כותב אותו מאפס. אתה מקבל אותו מוכן, קורא, מריץ, ומשנה שורה אחת.

הקוד

פתח את הקובץ הזה ב-VS Code (אני נתתי לך אותו בריפו, בתיקייה exercises/lesson-01/):

ב-Terminal — מתיקיית הריפו, הרץ:

Terminal 
code exercises/lesson-01/greet.py

זה התוכן של הקובץ:

greet.py 
# greet.py — סקריפט Python ראשון שמדבר איתך

name = input("מה השם שלך? ")
age_text = input("בן כמה אתה? ")

age = int(age_text)

print(f"שלום {name}!")
print(f"בעוד 10 שנים תהיה בן {age + 10}.")

קוראים שורה־שורה

שורה 1: # greet.py — ... — זה הערה. כל מה שמתחיל ב-# ש-Python מתעלם ממנו. ההערות בשבילך, לזכור מה הקוד עושה.

שורה 3: name = input("מה השם שלך? ") — כאן קורים שני דברים:

  • input(...) מציג טקסט ב-Terminal ומחכה שתקליד תשובה. כשאתה לוחץ Enter — הוא מחזיר את מה שהקלדת.
  • name = ... שומר את התשובה בתוך משתנה בשם name. משתנה זה כמו "תיבה עם שם" — מעכשיו אפשר לכתוב name בקוד וזה יתייחס לתשובה שלך.

שורה 4: אותו דבר — שואל גיל, שומר במשתנה age_text.

למה age_text ולא סתם age?

כי input() תמיד מחזיר טקסט (str), גם אם המשתמש הקליד מספר. אם תקליד 35 — Python יקבל את זה כמחרוזת "35", לא כמספר. כדי לעשות חשבון עם זה — צריך להמיר.

שורה 6: age = int(age_text) — הפונקציה int() ממירה מחרוזת למספר שלם. זה החלק הקריטי. בלי השורה הזו, השורה האחרונה הייתה זורקת שגיאה כי לא אפשר לעשות "35" + 10 (טקסט + מספר).

העיקרון: כל ערך יש לו טיפוס

ב-Python כל ערך שייך לטיפוס: str (טקסט), int (מספר שלם), float (מספר עם נקודה), bool (True/False). רוב הבאגים של מתחילים הם כי הטיפוס לא היה מה שחשבת. לא צריך לזכור את זה — צריך לזהות את זה כשתראה שגיאה.

שורות 8-9: print(f"שלום {name}!") — זה f-string. ה-f לפני המרכאות אומר ל-Python: "תחפש בתוך { } משתנים או ביטויים, ותחליף אותם בערכים שלהם". כלומר, אם name = "Shai", אז f"שלום {name}!" הופך ל-"שלום Shai!".

שים לב לביטוי {age + 10} — אפשר לשים בתוך הסוגריים גם חישוב, לא רק שם של משתנה. f-string הוא הדרך המודרנית והנקייה לבנות טקסט עם משתנים, ונשתמש בו לאורך כל הקורס.

פתח את greet.py ב-VS Code. שנה את ההודעה האחרונה מ-"בעוד 10 שנים תהיה בן..." ל-"בעוד 25 שנים תהיה בן...". שמור. הרץ שוב. ראה שהתוצאה השתנתה.

דוגמה 2 — מחלץ דומיין מ-URL

30 דקות
למה הדוגמה הזו

בעבודת SEO מטפלים כל הזמן בטקסט — URLs, כותרות, meta descriptions, פלט מ-API. הדוגמה הזו לוקחת URL ומחלצת ממנו את הדומיין. היא מדגימה את שתי הפעולות הכי שימושיות על מחרוזות: להחליף חלק אחד באחר, ולחתוך מחרוזת לחלקים. כשתבקש מ-Claude לעבד טקסט — סביר שהוא ישתמש בדיוק בדברים האלה.

הקוד

ב-Terminal — מהריפו, פתח את הקובץ:

Terminal 
code exercises/lesson-01/domain_extractor.py
domain_extractor.py 
# domain_extractor.py — מקבל URL ומחזיר רק את הדומיין

url = input("הכנס URL: ")

url = url.replace("https://", "").replace("http://", "")
url = url.replace("www.", "")

domain = url.split("/")[0]

print(f"הדומיין הוא: {domain}")

קוראים שורה־שורה

שורה 3: url = input(...) — מבקש URL מהמשתמש, שומר במשתנה url. זה כבר מוכר משיעור 2.

שורה 5: כאן יש משהו חדש —

דוגמה 
url = url.replace("https://", "").replace("http://", "")

זה נקרא מתודה — פונקציה ששייכת לערך. url.replace("https://", "") אומר: "קח את url, החלף כל מופע של "https://" במחרוזת ריקה """. החלפה במחרוזת ריקה = הסרה.

למה יש שתי קריאות ל-.replace אחת אחרי השנייה? כי URLs יכולים להיות עם https:// או http://. אנחנו מסירים את שניהם, ליתר ביטחון. שרשור המתודות עובד כי .replace מחזירה מחרוזת חדשה, ולתוצאה אפשר ישר לקרוא ל-.replace נוספת.

שורה 6: url = url.replace("www.", "") — אותו טריק. מסיר את www. אם הוא קיים.

העיקרון: .replace(X, "") = "תסיר את X"

זה דפוס שתפגוש כל הזמן בעבודה עם טקסט. רוצה להסיר רווחים? s.replace(" ", ""). רוצה לנקות מפסיקים? s.replace(",", ""). תזהה את הדפוס — לא תצטרך לזכור את התחביר, Claude יכתוב אותו.

שורה 8:

דוגמה 
domain = url.split("/")[0]

אחרי שהסרנו https:// ו-www., נשארת מחרוזת כמו beatport.com/genre/techno. שני דברים קורים בשורה הזו:

  • url.split("/") — חותך את המחרוזת לפי הסימן / ומחזיר רשימה: ["beatport.com", "genre", "techno"].
  • [0] — לוקח את האיבר הראשון ברשימה. ב-Python מתחילים לספור מ-0, אז [0] = ראשון, [1] = שני וכו'. במקרה שלנו, [0] הוא הדומיין.

שורה 10: print(f"הדומיין הוא: {domain}") — אותו f-string שהכרנו, מדפיס את התוצאה.

על slicing — בקצרה (לא צריך לזכור)

אם תסתכל בקוד שאחרים כתבו ב-Python, תראה לפעמים תחביר כמו s[0:5] או s[-4:]. זה נקרא slicing — לקיחת חתיכה מאמצע מחרוזת. הוא דומה ל-[0] שעשינו, רק שהוא לוקח טווח של תווים, לא תו בודד.

אני לא מצפה שתזכור את התחביר הזה. כשתצטרך — תבקש מ-Claude, הוא יכתוב לך. העיקר שתזהה: "אה, זה לוקח חתיכה ממחרוזת". זהו. ממשיכים.

הרצה

ב-Terminal — מתיקיית exercises/lesson-01, הרץ:

Terminal 
python3 domain_extractor.py

נסה עם כמה URLs שונים:

פלט ב-Terminal
הכנס URL: https://www.beatport.com/genre/techno
הדומיין הוא: beatport.com

הכנס URL: http://example.com/about
הדומיין הוא: example.com

הכנס URL: https://selected.co.il
הדומיין הוא: selected.co.il

המשימה שלך — שינוי קטן

הסקריפט הנוכחי לא מסיר פרמטרי URL כמו ?utm_source=.... אם תיתן לו https://beatport.com?utm_source=newsletter, התוצאה תהיה beatport.com?utm_source=newsletter — לא נקי.

המשימה: הוסף שורה אחת לפני שורת ה-split שתסיר הכל החל מסימן השאלה.

רמז

אותו טריק כמו split("/")[0] — רק עם ? במקום /. שורה אחת בלבד.

פתרון
domain_extractor.py (אחרי השינוי) 
url = input("הכנס URL: ")
url = url.replace("https://", "").replace("http://", "")
url = url.replace("www.", "")
url = url.split("?")[0]   # ← השורה החדשה: זורק את כל מה שאחרי סימן השאלה
domain = url.split("/")[0]
print(f"הדומיין הוא: {domain}")

אם הסתבכת — לא נורא. בעבודה האמיתית תבקש מ-Claude: "הסקריפט הזה לא מתמודד עם UTM params. תקן." זה היה הוקח 3 שניות.

דוגמה 3 — Python מדבר עם Claude ⭐

45 דקות
למה זה השיא של היום

עד עכשיו ראית Python מטפל בטקסט מקומי. החלק הזה מראה למה אתה בכלל לומד Python: בשביל לדבר עם Claude (או GPT, או כל AI אחר) דרך קוד — לא רק דרך הצ'אט. ברגע שהסקריפט הזה רץ אצלך, יש לך את התשתית לכל ה-vibe coding ולכל ה-AI agents שתבנה בקורס. שווה את ה-45 דקות.

4.1 קבלת API key מאנתרופיק

API key הוא "סיסמה" שמזהה את החשבון שלך כשאתה שולח בקשות ל-Claude מתוך קוד. בלי key — Claude לא יעבוד.

  1. היכנס ל-console.anthropic.com. הירשם עם המייל שלך (אפשר אותו מייל שאתה משתמש בו ל-Claude.ai).
  2. בתפריט בצד שמאל — Settings → Billing. הוסף $5 קרדיט (מינימום). זה יספיק לאלפי קריאות API קטנות לאורך הקורס.
  3. בתפריט — Settings → API Keys. לחץ Create Key. תן לו שם (למשל "vma-learning"). העתק את ה-key מיד — הוא יוצג רק פעם אחת.
חשוב — אל תפרסם את ה-key

API key הוא כמו סיסמה. אל תכניס אותו לקוד שאתה מעלה ל-GitHub. בקובץ .gitignore של הריפו הזה כבר רשום .env — קובץ שמיועד ל-keys ושלא יועלה לפומבי. אנחנו נשתמש בו עוד רגע.

4.2 התקנת חבילות Python

נשתמש בשתי חבילות: anthropic (ה-SDK הרשמי לדבר עם Claude) ו-python-dotenv (קוראת את ה-key מקובץ .env).

ב-Terminal — הרץ:

Terminal 
pip3 install anthropic python-dotenv

אם רואה הודעות על הורדה והתקנה ובסוף Successfully installed ... — מצוין.

מה זה pip?

pip הוא מנהל החבילות של Python — דרך להוריד ולהתקין קוד שאנשים אחרים כתבו. pip3 install anthropic אומר: "הורד את החבילה anthropic מ-PyPI (מאגר חבילות Python) והתקן אותה כדי שאוכל להשתמש בה בקוד שלי". זה כמו brew install, רק שזה ספציפי לחבילות Python.

4.3 קובץ .env — איפה שומרים את ה-key

ב-Terminal — מתיקיית exercises/lesson-01, צור קובץ .env:

Terminal

ask_claude.py 
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

question = input("מה לשאול את Claude? ")

response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=1024,
    system="אתה עוזר ידידותי שעונה בעברית, בצורה קצרה וברורה.",
    messages=[
        {"role": "user", "content": question}
    ]
)

answer = response.content[0].text
print("\nClaude עונה:\n")
print(answer)

4.5 קוראים את הקוד — לא לזכור, רק לזהות

שורות 1-3: import — "תביא לי את הכלים האלה". os מובנה ב-Python וקורא משתני סביבה. dotenv ו-anthropic אלה החבילות שהתקנו עכשיו.

שורה 5: load_dotenv() — קוראת את הקובץ .env וטוענת את המשתנים שבתוכו לזיכרון, כך שהקוד יוכל להגיע אליהם.

שורה 7: client = Anthropic(api_key=...) — יוצרים "לקוח" שיודע לדבר עם Claude. ה-os.environ["ANTHROPIC_API_KEY"] שולף את ה-key מתוך משתני הסביבה שנטענו ב-load_dotenv().

שורה 9: question = input(...) — מבקש מהמשתמש שאלה. מוכר.

שורות 11-18 — לב הסקריפט:

דוגמה 
response = client.messages.create(
    model="claude-haiku-4-5",
    max_tokens=1024,
    system="אתה עוזר ידידותי שעונה בעברית, בצורה קצרה וברורה.",

זאת הקריאה ל-Claude. ארבעה דברים שכדאי לזהות:

  • model — איזה מודל של Claude משתמשים. claude-haiku-4-5 הוא הזול והמהיר.
  • max_tokens — אורך תשובה מקסימלי. 1024 מספיק לתשובה בינונית.
  • system — "האישיות" של Claude לשיחה הזו. כאן אמרנו לו: "אתה עוזר ידידותי שעונה בעברית בצורה קצרה". זה מה שתשנה בקרוב.
  • messages — השיחה עצמה. כאן יש הודעה אחת מהמשתמש (השאלה שהוקלדה).

שורה 20: answer = response.content[0].text — התשובה חוזרת כאובייקט מורכב. הטקסט עצמו נמצא בעומק (content[0].text). זה תחביר שיופיע שוב ושוב — אל תזכור, תזהה.

שורות 21-22: מדפיסים את התשובה.

שאל אותו משהו. דוגמה:

פלט ב-Terminal
מה לשאול את Claude? תן לי 3 רעיונות לכותרות בלוג על SEO

Claude עונה:

1. "5 השגיאות הנפוצות שעושים ב-SEO בלי לדעת"

🎉 זה הרגע. Python הריץ את הקוד שלך, התחבר ל-Claude, קיבל תשובה, והדפיס. מעכשיו כל מה שאתה לומד בקורס יבנה על המקרה הזה.

המשימה שלך — שינוי ה-system prompt

השורה system="אתה עוזר ידידותי שעונה בעברית..." זה מה שמגדיר את ההתנהגות של Claude בשיחה. שנה אותה ל:

דוגמה 
system="אתה יועץ SEO בכיר עם 15 שנות ניסיון. אתה עונה בעברית, בצורה ישירה, ותמיד נותן 3 צעדים פרקטיים שאפשר לבצע מיד.",

שמור. הרץ שוב את הסקריפט עם אותה שאלה. שים לב איך התשובה שונה — אותו מודל, אותה שאלה, ה-system prompt שינה הכל.

למה זה הדבר הכי חשוב בשיעור

הקטע הזה — שינוי system prompt — הוא הסוד של 90% מבניית AI agents. כל סוכן AI שתבנה בקורס יתבסס בעצם על זה: מודל + system prompt טוב + הקשר מתאים = סוכן חכם. תרגיל בית: נסה 3 system prompts שונים על אותה שאלה. תראה כמה זה משנה.

שאלה 1

בקוד הבא:

קטע קוד
name = input("שם: ")
print(f"שלום {name.strip().lower()}")

אם המשתמש הקליד " SHAI " — מה יודפס?

תשובה

שלום shai. .strip() הסיר את הרווחים בקצוות → "SHAI". אחר כך .lower() הפך לאותיות קטנות → "shai". ה-f-string הדפיס.

שאלה 2

מה ההבדל בין שתי השורות הבאות?

קטע קוד
age = input("גיל: ")

תשובה

בשורה הראשונה, age הוא מחרוזת (str) — גם אם המשתמש הקליד מספר. בשורה השנייה, int() ממיר את הקלט למספר שלם, אז age הוא int. בלי ההמרה לא אפשר לעשות חשבון עם הערך הזה.
