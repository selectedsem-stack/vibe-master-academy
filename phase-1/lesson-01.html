Setup — מתקינים את הסביבה

1.5 שעות

מה זה Python בכלל?

Python היא שפת תכנות. כשאתה כותב קוד Python, אתה כותב הוראות למחשב שלך בשפה שיותר קלה לבני אדם לקרוא. הסקריפט עצמו הוא קובץ טקסט שמסתיים ב-.py.

דוגמה — איך נראה קוד Python:

hello.py 
print("Hello, Shai!")

זאת שורה אחת — היא מבקשת מהמחשב להדפיס את המילים האלה למסך. עוד מעט נכתוב אותה בעצמנו ב-VS Code ונריץ אותה ב-Terminal.

Terminal — לפני שמתחילים

ה-Terminal הוא חלון שבו אתה מקליד פקודות ישירות למחשב. 5 פקודות שתשתמש בהן בלי הפסקה:

פקודהמה היא עושהדוגמה
pwdמציג את התיקייה הנוכחיתpwd → /Users/shai
lsמציג את הקבצים בתיקייהls
cd <תיקייה>נכנס לתיקייהcd projects
cd ~חוזר לתיקיית הבית~ = home directory
mkdir <שם>יוצר תיקייה חדשהmkdir my-folder
טיפ חשוב

הסימן ~ תמיד מצביע על תיקיית הבית שלך (במאק: /Users/USERNAME). אז cd ~/projects = "לך לתיקיית projects שבתוך הבית שלי".

אם פקודה לא עובדת

קרא את הודעת השגיאה. command not found = הכלי לא מותקן. Permission denied = חסרות הרשאות — נסה להוסיף sudo לפני הפקודה, יבקש סיסמה.

יסודות Python

1 שעה
מאיפה אנחנו

סביבת Python מותקנת, VS Code מותקן, יש לנו hello.py שעובד. עכשיו נחבר את הכלים — איך לתת לקוד שלך לזכור ערכים, להבדיל בין סוגי נתונים, ולתקשר עם המשתמש.

2.1 משתנים — תיבות לערכים

משתנה הוא "תיבה" עם שם, שמכילה ערך. ב-Python יוצרים אחד עם סימן שווה =:

ב-VS Code — פתח קובץ חדש practice.py בתיקיית exercises/lesson-01/ והדבק:

practice.py 
name = "Shai"
age = 35
is_seo_head = True

print(name)
print(age)
print(is_seo_head)

ב-Terminal — מתיקיית exercises/lesson-01 הרץ:

Terminal 
python3 practice.py

צריך להופיע ב-Terminal:

פלט ב-Terminal
Shai
35
True
כללי שמות משתנים

שמות באנגלית קטנה, רווחים מוחלפים ב-underscore: first_name, total_traffic, is_active. לא מתחילים במספר (2cool ❌, cool2 ✅).

2.2 ארבעת הטיפוסים הבסיסיים

כל ערך ב-Python שייך לטיפוס מסוים. ארבעת הטיפוסים שתפגוש כל יום:

טיפוסדוגמהלמה זה
str"Shai"מחרוזת — טקסט (תמיד בגרשיים)
int35מספר שלם (בלי נקודה)
float3.14מספר עם נקודה עשרונית
boolTrue / Falseבוליאני — רק שני ערכים אפשריים

בשביל לבדוק טיפוס של ערך, יש פונקציה מובנית type():

ב-VS Code — הוסף ל-practice.py:

practice.py 
print(type(name))           # <class 'str'>
print(type(age))            # <class 'int'>
print(type(is_seo_head))    # <class 'bool'>
print(type(3.14))           # <class 'float'>

זה כלי שתשתמש בו כשמשהו לא עובד — לרוב הסיבה היא שהטיפוס לא מה שחשבת.

2.3 אופרטורים אריתמטיים

Python יודע לעשות חישובים. שש האופרטורים שתפגוש כל יום:

אופרטורשםדוגמהתוצאה
+חיבור10 + 313
-חיסור10 - 37
*כפל10 * 330
/חילוק10 / 33.333...
//חילוק שלם10 // 33
%שארית (modulo)10 % 31
מתי בכלל צריך % ו-// ?

% שימושי לבדיקה אם מספר מתחלק (למשל n % 2 == 0 — האם n זוגי). // שימושי כשרוצים תוצאה שלמה ולא עשרונית — למשל "כמה דקות שלמות יש ב-200 שניות": 200 // 60 = 3.

2.4 השוואות — ולמה הן מחזירות bool

אופרטורי השוואה משווים שני ערכים ומחזירים True או False:

אופרטורמשמעותדוגמה
==שווה (שים לב — שני סימני שווה)age == 35
!=לא שווהname != "Marcel"
>גדול מ-age > 18
<קטן מ-age < 100
>=גדול או שווהage >= 18
<=קטן או שווהtraffic <= 1000
הטעות הכי נפוצה

= זה השמה (תכניס לתוך), == זה השוואה (האם שווה?). age = 18 אומר "מעכשיו age הוא 18". age == 18 אומר "האם age שווה 18?" ומחזיר True/False.

2.5 print ו-input

print כבר השתמשת בו — הוא מדפיס ערך למסך. input עושה את ההפך: מחכה שהמשתמש יקליד משהו ב-Terminal, ומחזיר את מה שהוקלד כ-str.

ב-VS Code — פתח קובץ חדש interactive.py והדבק:

interactive.py 
name = input("מה השם שלך? ")
print("שלום,", name)

ב-Terminal — הרץ:

Terminal 
python3 interactive.py

הסקריפט מחכה. הקלד שם ולחץ Enter — אמור להופיע "שלום, <השם שלך>".

input תמיד מחזיר str

אם תקליד 35 בתגובה ל-input() — Python יקבל את זה כמחרוזת "35", לא כמספר. בשביל לבצע חישובים תצטרך להמיר עם int(). נראה את זה עוד שתי-שלוש דקות.

2.6 המרת טיפוסים

שלוש פונקציות שימושיות להמרה:

  • int(x) — ממיר מחרוזת/float למספר שלם
  • float(x) — ממיר מחרוזת/int למספר עשרוני
  • str(x) — ממיר כל דבר למחרוזת

ב-VS Code — צור age_check.py והדבק:

age_check.py 
age_text = input("מה הגיל שלך? ")   # input מחזיר str
age = int(age_text)                  # ממירים ל-int
print("בעוד 10 שנים תהיה בן", age + 10)

אם לא היינו עושים int() ומנסים age_text + 10 — Python היה זורק שגיאה TypeError כי לא אפשר לחבר מחרוזת ומספר.

2.7 f-strings — הדרך הנקייה לבנות הודעות

במקום print("שלום,", name), יש דרך יותר נקייה — f-string:

דוגמה 
name = "Shai"
age = 35

print(f"שלום, {name}!")
print(f"אתה בן {age}, בעוד 10 שנים תהיה בן {age + 10}")

ה-f לפני הגרשיים אומר ל-Python: "זה לא טקסט רגיל, תחפש משתנים או ביטויים בתוך {} והחלף אותם בערכים". זה הסטנדרט המודרני — נשתמש בו לאורך הקורס.

2.8 הערות בקוד

הערות הן טקסט בקוד שה-Python מתעלם ממנו. הן בשבילך, לזכור מה הקוד עושה:

דוגמה 
# זאת הערה — לא משפיעה על ההרצה
name = "Shai"   # זאת גם הערה, אחרי קוד
הרגל מצוין

הוסף הערה לפני כל שורה לא טריוויאלית. עוד שנה תודה לעצמך.

2.9 אינדנטציה — הסוד של Python

ב-Python, רווחים בתחילת השורה הם משמעותיים. שלא כמו בשפות אחרות שמשתמשות ב-{ }, Python מסתמך על אינדנטציה כדי להבין מה שייך לאן.

לדוגמה, ב-if:

דוגמה 
age = 18
if age >= 18:
    print("מבוגר")          # 4 רווחים בהתחלה
    print("יכול להצביע")     # אותם 4 רווחים — באותו הבלוק
print("זה לא תלוי בגיל")    # אין רווחים — מחוץ לבלוק
שגיאה ראשונה של כל מתחיל ב-Python

IndentationError. אם תקבל את זה — תבדוק את הרווחים. השתמש תמיד ב-4 רווחים, לא tabs. ב-VS Code, אם תלחץ Tab, הוא יהפוך אותו לרווחים אוטומטית (זה הסטנדרט).

תרגיל מהיר — בדוק את עצמך

לפני שעוברים הלאה, נסה לבד את שלושת התרגילים. תכתוב את התשובות לעצמך לפני שתסתכל בפתרון.

תרגיל 1 — מה הטיפוס?

איזה טיפוס הם הערכים הבאים: "35", 35, 35.0, True?

פתרון
  • "35" → str (יש גרשיים)
  • 35 → int (מספר שלם, בלי נקודה)
  • 35.0 → float (יש נקודה)
  • True → bool
תרגיל 2 — מחשבון רווחי

כתוב סקריפט profit.py ששואל מהמשתמש כמה רווחו בחודש שעבר ובחודש הזה, ומדפיס את ההפרש.

פתרון
profit.py 
last_month = int(input("רווח חודש שעבר: "))
this_month = int(input("רווח חודש הזה: "))
diff = this_month - last_month
print(f"ההפרש: {diff} ש\"ח")

שים לב ל-int() — בלעדיו תקבל TypeError כי input() מחזיר str.

תרגיל 3 — ברכת יום הולדת

כתוב סקריפט birthday.py ששואל שם וגיל, ומדפיס: "שלום <שם>, מזל טוב! בעוד שנה תהיה בן/בת <גיל+1>." — השתמש ב-f-string.

פתרון
birthday.py 
name = input("שם: ")
age = int(input("גיל: "))
print(f"שלום {name}, מזל טוב! בעוד שנה תהיה בן {age + 1}.")
3

מחרוזות לעומק

1.5 שעות
למה זה החלק הכי חשוב בשיעור

בעבודת SEO וב-vibe coding באופן כללי, רוב הזמן אתה מטפל בטקסט — URLs, כותרות, meta descriptions, תוצאות מ-API. ב-Python, כל טקסט הוא str. החלק הזה מלמד אותך לעבד מחרוזות לעומק: לחתוך, להחליף, לפצל. כל הכלים בתרגיל הסופי של השיעור מגיעים מכאן.

3.1 יצירת מחרוזות — שלוש דרכים

אפשר ליצור מחרוזת ב-3 אופנים:

דוגמה 
a = "Shai Berdugo"           # גרשיים כפולים
b = 'Head of SEO'            # גרש בודד
c = """שורה ראשונה
שורה שנייה
שורה שלישית"""              # שלוש מרכאות — לטקסט רב-שורתי

print(a)
print(b)
print(c)
מתי להשתמש בכל אחת?

גרשיים כפולים ("...") — ברירת המחדל. גרש בודד ('...') — אם המחרוזת מכילה גרשיים כפולים בתוכה. למשל 'הוא אמר "שלום"'. שלוש מרכאות — לטקסט שמתפרש על כמה שורות (כמו prompts ל-Claude).

3.2 חיבור וכפל מחרוזות

אפשר לחבר שתי מחרוזות עם +, ולכפול מחרוזת במספר:

דוגמה 
first = "Shai"
last = "Berdugo"
full = first + " " + last
print(full)              # Shai Berdugo

line = "-" * 30
print(line)              # ------------------------------

שים לב — first + " " + last דורש להוסיף את הרווח באופן ידני. בדרך כלל יותר נקי להשתמש ב-f-string: f"{first} {last}".

3.3 אורך מחרוזת — len()

הפונקציה len(s) מחזירה את מספר התווים במחרוזת:

דוגמה 
name = "Shai"
print(len(name))           # 4

empty = ""
print(len(empty))          # 0

sentence = "Head of SEO"
print(len(sentence))       # 11 (כולל הרווחים!)

שימוש מעשי: לבדוק אם משתמש הזין משהו (if len(name) == 0), או לבדוק שכותרת SEO לא חורגת מהאורך המומלץ של 60 תווים.

3.4 אינדקסים — לקבל תו ספציפי

במחרוזת ב-Python, לכל תו יש אינדקס (מספר סידורי). חשוב: הספירה מתחילה מ-0, לא מ-1.

דוגמה 
s = "Python"
#    P y t h o n
#    0 1 2 3 4 5    ← אינדקסים חיוביים
#   -6-5-4-3-2-1    ← אינדקסים שליליים

print(s[0])    # P  — התו הראשון
print(s[1])    # y  — השני
print(s[5])    # n  — השישי (האחרון)
print(s[-1])   # n  — האחרון, דרך קצרה
print(s[-2])   # o  — לפני האחרון
למה להתחיל מ-0?

זה מוסכמה בכל שפות התכנות כמעט. תרגל אותה — בהתחלה זה מבלבל, אחרי שבוע זה הופך לטבעי. אינדקסים שליליים ((-1, -2, ...) שימושיים מאוד — תמיד מספרים מהסוף, גם אם לא יודעים את האורך.

3.5 חיתוך (slicing) — לקחת קטע

אם אתה רוצה קטע מהמחרוזת (לא תו בודד), השתמש בתחביר s[start:end] — נקרא slicing:

דוגמה 
url = "https://beatport.com"

print(url[0:8])      # https://
print(url[8:])       # beatport.com   (מ-8 עד הסוף)
print(url[:8])       # https://       (מההתחלה עד 8)
print(url[-4:])      # .com           (4 התווים האחרונים)
print(url[-4:-1])    # .co            (לא כולל האחרון)
חשוב לזכור

בחיתוך, start כלול אבל end לא כלול. כלומר s[0:3] מחזיר את האינדקסים 0, 1, 2 — בלי 3. כשמשמיטים את start זה מתחיל מ-0, כשמשמיטים את end זה ממשיך עד הסוף.

3.6 מתודות גודל ורווח — lower / upper / strip

מתודה (method) זה כמו פונקציה ששייכת לערך. הסינטקס: value.method(). שלוש המתודות הראשונות שתשתמש בהן ב-99% מהמקרים שמטפלים ב-input מהמשתמש:

דוגמה 
name = "  Shai Berdugo  "

print(name.lower())     # "  shai berdugo  "  — הכל קטן
print(name.upper())     # "  SHAI BERDUGO  "  — הכל גדול
print(name.strip())     # "Shai Berdugo"      — בלי רווחים בקצוות

# כל אחת מחזירה מחרוזת חדשה — name עצמו לא משתנה
print(name)             # "  Shai Berdugo  "  — עדיין כמו שהיה
חשוב — מחרוזות ב-Python הן "immutable"

קריאה ל-name.lower() לא משנה את name. היא מחזירה מחרוזת חדשה. אם אתה רוצה לשמור את התוצאה — צריך להציב חזרה: name = name.lower().

למה זה חשוב? כשמשתמש כותב לך URL, יכול להיות שהוא יקליד " Beatport.COM ". אם תשווה לזה "beatport.com" — לא יהיה matching. הפתרון: url.strip().lower() מנרמל את הקלט.

3.7 .replace(old, new) — המתודה החשובה ביותר

המתודה .replace מחזירה מחרוזת חדשה עם החלפה של כל מופע של old ב-new:

דוגמה 
s = "hello world hello"

print(s.replace("hello", "hi"))     # "hi world hi"   — מחליפה את כל המופעים
print(s.replace("world", "Python")) # "hello Python hello"

# החלפה במחרוזת ריקה = הסרה
s2 = "https://beatport.com"
print(s2.replace("https://", ""))   # "beatport.com"

# הסרת רווחים מכל מקום (לא רק מהקצוות כמו strip):
s3 = "S h a i"
print(s3.replace(" ", ""))          # "Shai"
הטריק: החלפה בריק = הסרה

זה מאוד שימושי. אם אתה רוצה להסיר משהו ממחרוזת — .replace("מה_להסיר", ""). למשל לנקות URL מ-"https://", להסיר "www.", להסיר תו ספציפי.

3.8 .split(separator) — לחתוך לחתיכות

המתודה .split(separator) מחלקת את המחרוזת לחתיכות לפי תו או מחרוזת מפריד. היא מחזירה list (מבנה נתונים שנלמד עליו לעומק ב-Lesson 2, אבל יש לנו כאן הצצה).

דוגמה 
csv = "Shai,Marcel,Niv"
parts = csv.split(",")
print(parts)            # ['Shai', 'Marcel', 'Niv']

# גישה לחתיכה ספציפית עם אינדקס:
print(parts[0])         # 'Shai'    — הראשון
print(parts[1])         # 'Marcel'
print(parts[-1])        # 'Niv'     — האחרון

# חיתוך URL לפי סלאש:
url = "beatport.com/genre/techno"
url_parts = url.split("/")
print(url_parts)        # ['beatport.com', 'genre', 'techno']
print(url_parts[0])     # 'beatport.com'   ← הדומיין!
מה זה list בעצם?

List זה סוגריים מרובעים [...] עם איברים מופרדים בפסיק. הפעולה list[0] לוקחת את האיבר הראשון. ההצצה הזו מספיקה כדי לפתור את התרגיל בסוף השיעור. את הכל לעומק (יצירה, הוספה, הסרה, lambda, sorted) נלמד ב-Lesson 2.

3.9 שרשור מתודות — Method chaining

כיוון שכל מתודה מחזירה מחרוזת חדשה, אפשר לקרוא למתודה הבאה ישר על התוצאה — שרשור:

דוגמה 
messy = "  HTTPS://BeatPort.COM  "

# שלוש מתודות, אחת אחרי השנייה
clean = messy.strip().lower().replace("https://", "")
print(clean)            # "beatport.com"

הקוד הזה קריא יותר מאשר 3 שורות נפרדות. השרשור עובד כי כל מתודה מחזירה מחרוזת — והמתודה הבאה רצה על התוצאה.

איך לקרוא שרשור

קרא משמאל לימין (כי הקוד LTR): "קח את messy, תפעיל .strip() (תסיר רווחים מקצוות), אחר כך .lower() (תהפוך לקטנות), אחר כך .replace (תסיר את ה-https://). זאת התוצאה."

תרגיל 3.A — נירמול קלט

כתוב סקריפט normalize.py ששואל את המשתמש לשם שלו, ומדפיס את השם בלי רווחים בקצוות ובאותיות קטנות. למשל אם הקלט הוא " Shai Berdugo ", הפלט צריך להיות "shai berdugo".

פתרון
normalize.py 
name = input("מה השם שלך? ")
clean_name = name.strip().lower()

שים לב לשרשור .strip().lower() — strip הסיר את הרווחים, lower הפך לקטנות.

תרגיל 3.B — הסרת סיומת קובץ

כתוב סקריפט filename.py ששואל שם קובץ עם סיומת (למשל "report.pdf") ומדפיס רק את השם בלי הסיומת ("report"). רמז: השתמש ב-.split(".") וקח את האיבר הראשון.

פתרון
filename.py 
filename = input("שם קובץ: ")
parts = filename.split(".")
name_only = parts[0]
print(f"השם בלי סיומת: {name_only}")

גרסה מקוצרת בשרשור: name_only = filename.split(".")[0]

תרגיל 3.C — נירמול URL

כתוב סקריפט url_normalize.py ששואל URL ומדפיס אותו בלי https:// או http:// בהתחלה. למשל "https://beatport.com" → "beatport.com". (שים לב — זה רק חלק ממה שהתרגיל הגדול ידרוש.)

פתרון

שתי קריאות .replace בשרשור. הראשונה מסירה את https://, השנייה את http:// (לכל מקרה שאחד מהם קיים).

המטרה

צור קובץ חדש: exercises/lesson-01/domain_extractor.py. המטרה — סקריפט שמקבל URL מהמשתמש ומדפיס רק את הדומיין.

דוגמה לתפעול
הכנס URL: https://www.beatport.com/genre/techno
הדומיין הוא: beatport.com

אל תפתח את Claude עדיין. נסה לבד 30 דקות לפחות. אם תקעת — אז תפתח את הרמזים.

רמז 1 — חשוב על השלבים

ה-URL הוא https://www.beatport.com/genre/techno. מה צריך להסיר כדי להישאר עם beatport.com?

רמז 2 — איך מסירים חלק ממחרוזת

מחרוזות ב-Python יש להן מתודה .replace(old, new) שמחליפה חלק אחד באחר. אם רוצים להסיר משהו — מחליפים אותו במחרוזת ריקה "".

רמז 3 — איך חותכים את הסוף

אחרי שהסרת את https:// ו-www., נשארת לך מחרוזת כמו beatport.com/genre/techno. מתודה .split("/") מחזירה רשימה כשהיא חותכת בסלאש. האיבר הראשון ברשימה (אינדקס [0]) הוא הדומיין.

פתרון מלא (רק אחרי שניסית!)
domain_extractor.py 
# domain_extractor.py
url = input("הכנס URL: ")

# מסירים את הפרוטוקול
url = url.replace("https://", "").replace("http://", "")

# מסירים www
url = url.replace("www.", "")

# חותכים בסלאש הראשון
domain = url.split("/")[0]

print("הדומיין הוא:", domain)

הרץ עם 3 URLs שונים כדי לראות שזה עובד.
