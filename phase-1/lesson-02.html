Lesson 2 — Data + AI

רקע — איך data נראה ב-Python

30 דקות
מאיפה אנחנו

בשיעור 1 ראינו משתנים בודדים: name = "Shai", age = 35. אבל בעבודה אמיתית יש לך לא משתנה אחד — יש לך אוסף של דברים. רשימת לקוחות, רשימת מתחרים, רשימת מילות מפתח, רשימת פוסטים. שני המבנים שאתה תפגוש בכל יום ב-Python (ובכל AI agent) הם list ו-dict.

1.1 List — רשימה

list הוא אוסף סדור של ערכים. סוגריים מרובעים, איברים מופרדים בפסיק:

דוגמה
clients = ["מנגו", "סופר ביץ'", "קליניקת רימון"]

print(clients[0])      # 'מנגו'  — האיבר הראשון (ספירה מ-0!)
print(clients[-1])     # 'קליניקת רימון' — האחרון
print(len(clients))    # 3 — כמה איברים יש
העיקרון — list = "אוסף מסודר"

הספירה מתחילה מ-0. list[0] הוא הראשון, list[-1] הוא האחרון. כשתראה my_list[2] בקוד שClaude כתב — תזהה: "אה, האיבר השלישי ברשימה הזאת". זה כל הידע שאתה צריך.

1.2 Dict — מילון

dict הוא אוסף של זוגות key-value. סוגריים מסולסלים, כל זוג מופרד בפסיק:

דוגמה
client = {
    "name": "מנגו",
    "traffic": 45000,
    "industry": "fashion",
    "has_blog": True,
}

print(client["name"])       # 'מנגו'
print(client["traffic"])    # 45000
print(client["has_blog"])   # True

בניגוד ל-list, כאן לא משתמשים במספרים אלא ב-שמות (keys). מה ש-list[0] הוא ל-list, dict["name"] הוא ל-dict.

למה זה החלק הכי חשוב — list של dicts

הצורה הנפוצה ביותר של data ב-Python (וב-99% מ-APIs בעולם, כולל Claude) היא list של dicts: רשימה של דברים, שלכל אחד יש שדות. כמו טבלה ב-Excel. הקובץ clients.json שלנו הוא בדיוק זה.

הצורה הקלאסית
clients = [
    {"name": "מנגו",          "traffic": 45000, "industry": "fashion"},
    {"name": "סופר ביץ'",      "traffic": 32000, "industry": "ecommerce"},
    {"name": "קליניקת רימון",  "traffic": 12000, "industry": "healthcare"},
]

clients[1]["name"]      # 'סופר ביץ'' — list ראשון, אז dict
clients[0]["traffic"]   # 45000

clients[1]["name"] נקרא משמאל לימין: "קח את clients, את האיבר באינדקס 1, ושלוף ממנו את ה-key בשם name". זה דפוס שתראה כל יום בעבודה עם AI.

תשובה

קטע קוד

    {"title": "SEO ב-2026", "views": 1200},

2.3 קוראים שורה־שורה

שורה 1: import json — מביא את הספרייה json של Python. יודעת לקרוא ולכתוב קבצי JSON.

שורות 3-4 — פתיחת הקובץ:

קטע קוד
with open("clients.json", encoding="utf-8") as f:
    clients = json.load(f)
  • open(...) פותח את הקובץ. encoding="utf-8" קריטי לטקסט עברי.
  • with ... as f: — context manager. מבטיח שהקובץ ייסגר אוטומטית בסוף. תמיד משתמשים בזה.
  • clients = json.load(f) — קורא את התוכן וממיר ל-list/dict של Python.

שורה 6 — המיון:

sorted
sorted_clients = sorted(clients, key=lambda c: c["traffic"], reverse=True)
  • sorted(clients, ...) — ממיין את הרשימה.
  • key=lambda c: c["traffic"] — איך למיין. ה-lambda זאת "פונקציה מקוצרת": "קח לקוח c, החזר את c["traffic"]". sorted ישתמש בערך הזה להחליט מי קודם.
  • reverse=True — מהגדול לקטן.
על lambda — לא צריך לזכור

lambda c: c["traffic"] זה תחביר מקוצר לפונקציה חד-שורתית. אתה לא צריך לכתוב lambda בעצמך. שתזהה אותו ב-Claude → "אה, זאת פונקציה קטנה שמושכת שדה מסוים". זה כל מה שצריך.

שורה 7: top_5 = sorted_clients[:5] — slicing מהשיעור הקודם. לוקח את 5 הראשונים.

שורות 9-11 — הלולאה:

for loop
for client in top_5:
    print(f"  {client['name']:<20} ...")

for ... in ... = הלולאה הבסיסית של Python. קורא בקול: "לכל client ברשימה top_5, בצע את הקוד הבא". בכל איטרציה המשתנה client מקבל את האיבר הבא.

הסימנים :<20 ו-:>8, הם format specifiers ל-f-string — רוחב עמודה, יישור, וירגולים בין אלפים. לזהות, לא לזכור.

import json


def analyze_client(client: dict) -> dict:
    """
    מנתח לקוח אחד ומחזיר דוח SEO מבוסס-כללים.

    מקבל:
        client (dict): לקוח עם שדות name, traffic, conversion_rate,
                       has_blog, last_audit_days, וכו'.

    מחזיר:
        dict עם:
            - client_name (str)
            - score (int 0-100)
            - recommendations (list של strings)
    """
    score = 50
    recommendations = []

    if client["traffic"] >= 50000:
        score += 25
    elif client["traffic"] >= 20000:
        score += 15
    elif client["traffic"] >= 10000:
        score += 5
    else:
        recommendations.append("תנועה אורגנית נמוכה — נדרשת אסטרטגיית תוכן + בניית סמכות תחומית")

    if client["has_blog"]:
        score += 10
    else:
        recommendations.append("אין בלוג. הקמת בלוג עם 2-4 פוסטים בחודש יכולה להעלות תנועה אורגנית ב-30-50% תוך 6 חודשים")

    if client["conversion_rate"] >= 3.5:
        score += 15
    elif client["conversion_rate"] < 1.5:
        recommendations.append(f"אחוז המרה נמוך ({client['conversion_rate']}%) — בחן UX, CTAs ו-trust signals")

    if client["last_audit_days"] > 90:
        recommendations.append(f"לא בוצעה ביקרת SEO ב-{client['last_audit_days']} ימים — בקרת technical SEO חיונית")
    elif client["last_audit_days"] <= 30:
        score += 5

    return {
        "client_name": client["name"],
        "score": min(score, 100),
        "recommendations": recommendations,
    }


with open("clients.json", encoding="utf-8") as f:
    clients = json.load(f)

for c in clients:
    report = analyze_client(c)
    print(f"\n📊 {report['client_name']:<20} ציון: {report['score']}/100")
    if report["recommendations"]:
        for rec in report["recommendations"]:
            print(f"   • {rec}")
    else:
        print("   ✓ אין המלצות — הלקוח במצב טוב")

3.2 קוראים את הפונקציה

החתימה (signature):

def
def analyze_client(client: dict) -> dict:

ארבעה דברים לזהות:

  • def — "פונקציה מתחילה כאן".
  • analyze_client — שם הפונקציה.
  • (client: dict) — מקבלת פרמטר בשם client, מסוג dict. ה-: dict הוא type hint — רמז לטיפוס.
  • -> dict — מחזירה dict. עוד type hint על הפלט.

ה-docstring (הטקסט בתוך """...""") הוא תיעוד של הפונקציה. כלי AI קוראים אותו כדי להבין מה היא עושה. תרגיל לעצמך: כשתבקש מ-Claude לכתוב פונקציה — בקש שיכלול docstring. זה משפר קריאות פי 10.

גוף הפונקציה:

  • if / elif / else — בדיקות תנאי. רק אחד מהבלוקים רץ. סדר חשוב — Python בודק אחד אחרי השני ועוצר בראשון שמתקיים.
  • score += 25 — קיצור ל-score = score + 25.
  • recommendations.append(...) — מתודה של list שמוסיפה איבר לסוף. אחרי כמה .append, יש לך רשימה עם כמה המלצות.

השורה החשובה ביותר — return:

return
return {
    "client_name": client["name"],
    "score": min(score, 100),
    "recommendations": recommendations,
}

מחזיר dict עם שלושה שדות. min(score, 100) = "הקטן יותר מבין השניים" — מוודא שהציון לא יעבור 100.

3.3 איך קוראים לפונקציה

קריאה
for c in clients:
    report = analyze_client(c)
    print(f"\n📊 {report['client_name']:<20} ציון: {report['score']}/100")

בכל איטרציה: c הוא לקוח אחד, analyze_client(c) מחזיר dict עם הניתוח, ואז משתמשים בערכים שלו. זה הדפוס.

למה זה חשוב לעבודה שלך

אם תבקש מ-Claude "תכתוב לי כלי שבודק 10 דברים על כל לקוח" — הוא יחזיר פונקציה כמו זאת. השפה שלו תהיה: "analyze_client takes a client dict and returns a report". אם אתה יודע לקרוא את החתימה, אתה יודע איך לשלב את הקוד שלו בעבודה שלך.

תוספת
    if client["industry"] == "ecommerce" and not client["has_blog"]:

import json
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv("../lesson-01/.env")

claude = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])


def get_seo_advice(client_data: dict) -> str:
    """
    שולח את נתוני הלקוח ל-Claude ומחזיר 3 המלצות SEO ספציפיות בעברית.
    """
    prompt = f"""נתוני לקוח SEO:
- שם: {client_data['name']}
- דומיין: {client_data['domain']}
- תנועה אורגנית חודשית: {client_data['traffic']:,} ביקורים
- תעשייה: {client_data['industry']}
- אחוז המרה: {client_data['conversion_rate']}%
- יש בלוג: {'כן' if client_data['has_blog'] else 'לא'}
- מילת מפתח עיקרית: {client_data['top_keyword']}

תן 3 המלצות SEO ספציפיות לעסק הזה — ממוקדות, פרקטיות, מבוססות על הנתונים.
שורה אחת לכל המלצה. בלי הקדמות, בלי תרגומים, רק 3 שורות עם המלצות."""

    response = claude.messages.create(
        model="claude-haiku-4-5",
        max_tokens=512,
        system="אתה יועץ SEO בכיר עם 15 שנות ניסיון בשוק הישראלי. אתה עונה בעברית, בצורה תכליתית.",
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

posts = [

מה יודפס?

תשובה

שאלה 2

איך תקרא לפונקציה? מה היא תחזיר אם נעביר {"name": "X", "traffic": 80000, "has_blog": True}?

קטע קוד
def is_strong(client: dict) -> bool:
    return client["traffic"] > 50000 and client["has_blog"]
תשובה

קוראים: is_strong({"name": "X", "traffic": 80000, "has_blog": True}). מחזירה True — traffic = 80000 > 50000 וגם has_blog = True. שני התנאים מתקיימים → and = True.

שאלה 3

מה ההבדל בין json.load(f) ל-json.dump(data, f)?

תשובה

json.load(f) — קורא קובץ JSON ומחזיר Python (list/dict). מצב read.
json.dump(data, f) — כותב Python (list/dict) לקובץ כ-JSON. מצב write.
טריק לזכור: load = מתוך הקובץ אל הזיכרון. dump = מהזיכרון אל הקובץ.
