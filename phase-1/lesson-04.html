Lesson 4 — n8n + אוטומציה (Capstone)

סיימת 3 שיעורים של Python ויודע לקרוא, להריץ ולשנות קוד. היום אתה מגלה שאת אותם דברים אפשר לעשות בלי שורת קוד אחת — דרך n8n. זה הסיום של Phase 1: אתה תבנה 2 workflows מוכנים יחד איתי, ו-workflow שלישי לבד. החיבור הקסום: Python ל-logic מותאם, n8n לחיבור בין מערכות.

בסוף השיעור תדע
  • מה זה n8n, מתי משתמשים בו, ומה ההבדל מ-Zapier
  • לבנות workflow ב-n8n עם nodes, triggers, ו-credentials
  • לחבר n8n ל-Gmail, ל-HTTP חיצוני, ול-Claude API דרך node ייעודי
  • לבנות webhook: URL שמופעל מבחוץ ומריץ workflow
  • להריץ workflow שלם שמקבל שאלה → שולח ל-Claude → מחזיר תשובה למייל — בלי שורת Python אחת
n8n cloud workflows webhooks Gmail HTTP Request node Claude via n8n curl testing Phase 1 Capstone
למה צריך גם Python וגם n8n

הם לא מתחרים — הם משלימים. n8n מצוין לחיבור בין שירותים (Slack, Gmail, Notion, Stripe, Claude...) — workflows שאם תכתוב ב-Python יקח לך שעות. Python מצוין ל-logic מותאם — תנאים מורכבים, עיבוד nuanced, ניתוח data. בקורס נשתמש בשניהם — n8n כשהשירותים הם הכוכבים, Python כשה-logic הוא הכוכב.

Setup — n8n cloud + מושגי יסוד

30 דקות

1.1 n8n vs Zapier vs Make — ההבדל ב-3 שורות

כליחוזקלמי
Zapierהכי קל, הכי הרבה אינטגרציותnon-dev, פשטות מעל גמישות
Makevisual + יחס מחיר/ערך טובmiddle ground
n8nגמיש כמו קוד, יש self-hosted חינמיvibe coders, אנשי טכני

למה אנחנו ב-n8n: כי בעתיד תרצה להריץ workflows מסובכים עם הרבה תנאים ולוגיקה — וגם תרצה אופציה לרוץ אצלך self-hosted בלי תשלום. n8n היא הבחירה של vibe coders.

1.2 הרשמה ל-n8n cloud

  1. כנס ל-n8n.io/cloud והירשם עם המייל שלך.
  2. בחר ב-Free Trial (14 ימים, ללא כרטיס אשראי). אחרי הניסיון אפשר להמשיך עם plan חינמי מוגבל, או לעבור ל-self-hosted (חינם לחלוטין, נטפל בזה ב-Phase 2 אם נצטרך).
  3. אחרי שתאשר את המייל — תקבל URL ייחודי כמו https://shai.app.n8n.cloud. שמור אותו.

1.3 4 מושגי היסוד של n8n

מושגמה זה
Workflow"זרימת עבודה" — סדרה של פעולות שמחוברות בחיצים. רץ מתחילה לסוף.
Nodeצעד יחיד בתוך workflow. כל node עושה דבר אחד: שליחת מייל, קריאה ל-API, פילטור, וכו'.
Triggerה-node הראשון. מה מפעיל את ה-workflow? Manual trigger (כפתור), Webhook (קריאה מבחוץ), Schedule (Cron), וכו'.
Credentialפרטי גישה מוצפנים לשירות חיצוני. מאוחסנים מוצפנים ב-n8n. כל node שמתחבר לשירות חיצוני (Gmail, Claude) משתמש ב-credential.
המנטלי המודל

תחשוב על workflow כעל pipeline — בדיוק כמו ה-pipeline של topic_researcher.py משיעור 3 (Wikipedia → Claude → output). בדיוק אותם רעיונות, רק שב-n8n זה גרירה ושחרור ולא קוד.

Workflow 1 — Hello Email (workflow ראשון)

30 דקות
מה נבנה

workflow פשוט עם 2 nodes: Manual Trigger → Gmail Send. תלחץ כפתור — אתה מקבל מייל. זה ה-"Hello World" של n8n.

2.1 יצירת workflow חדש

  1. ב-n8n cloud, לחץ + New Workflow בצד שמאל.
  2. אתה רואה canvas ריק. בצד ימין יש לוח nodes לבחירה.

2.2 הוספת Manual Trigger

  1. לחץ + Add first step במרכז המסך.
  2. חפש "Manual Trigger", בחר אותו.
  3. ה-node מופיע ב-canvas. זה הצעד הראשון שלך.

Manual Trigger מופעל כשאתה לוחץ Execute Workflow — שימושי לפיתוח ובדיקות.

2.3 הוספת Gmail node

  1. לחץ על העיגול הקטן בצד ה-Manual Trigger (output).
  2. חפש "Gmail", בחר.
  3. בחר ב-action Send Email / Send a message.

n8n יבקש credential. לחץ + Create new credential ובחר "Sign in with Google". אשר ב-popup של Google. החזרה ל-n8n — ה-credential מאוחסן ומוצפן.

חשוב — הרשאות Gmail

Google ידרוש ממך אישור — בחר את אותו חשבון Gmail שאתה משתמש בו. צריך לאשר גישה מלאה לשליחת מיילים כדי שזה יעבוד.

2.4 הגדרת המייל

במאפייני ה-Gmail node:

  • To: המייל שלך (תקבל את המייל לעצמך)
  • Subject: Hello from n8n!
  • Email Type: Text (פשוט יותר ממולטימדיה)
  • Message: This is my first n8n workflow. 🎉

2.5 הרצה ובדיקה

  1. לחץ Execute Workflow בתחתית.
  2. תראה את ה-nodes הופכים לירוקים אחד אחרי השני.
  3. בדוק את ה-Gmail שלך — אמור להיות שם מייל חדש.
Troubleshooting
  • אם ה-Gmail node אדום — לחץ עליו, תקרא את הודעת השגיאה. בדרך כלל זה credential שלא אומת.
  • אם המייל לא הגיע — בדוק ב-Spam folder.
  • אם הכל ירוק אבל אין מייל — וודא שכתבת את הכתובת נכון ב-To.

שמור את ה-workflow (Cmd+S או הכפתור Save). תן לו שם: hello-email.

⭐ Workflow 2 — AI Assistant דרך Webhook

60 דקות
למה זה השיא של היום

זה בדיוק אותה לוגיקה של topic_researcher.py משיעור 3 — מקבל קלט, שולח ל-Claude, מחזיר תשובה. אבל הפעם:

  • בלי קוד — רק גרירה ושחרור
  • מופעל מבחוץ דרך webhook URL שתוכל לקרוא לו מ-curl, מהדפדפן, או מ-Slack
  • שולח תשובה למייל — אוטומציה אמיתית

אחרי שתבין את הדפוס הזה, אתה תוכל לחבר Claude לכל שירות בעולם.

3.1 יצירת workflow חדש

צור workflow חדש. שם: ai-assistant.

3.2 Node 1 — Webhook Trigger

  1. הוסף Webhook כ-trigger (במקום Manual).
  2. במאפיינים: HTTP Method = POST. Path = ai-assistant.
  3. תקבל 2 URLs:
    • Test URL — לבדיקות (פעיל רק כשה-workflow פתוח)
    • Production URL — לעבודה אמיתית (פעיל אחרי שתלחץ Activate)

העתק את ה-Test URL לפנקס — תשתמש בו בעוד רגע.

3.3 Node 2 — HTTP Request ל-Claude

הוסף HTTP Request אחרי ה-Webhook. הגדרות:

  • Method: POST
  • URL: https://api.anthropic.com/v1/messages
  • Authentication: Generic Credential Type → Header Auth. תיצור credential חדש:
    • Name: x-api-key
    • Value: ה-API key שלך מאנתרופיק (אותו אחד מ-.env בשיעור 1)
  • Send Headers: Yes. הוסף:
    • anthropic-version : 2023-06-01
    • content-type : application/json
  • Send Body: Yes. Body Content Type: JSON.
  • JSON:
    JSON Body
    {
  "model": "claude-haiku-4-5",
  "max_tokens": 512,
  "system": "אתה עוזר ידידותי שעונה בעברית בצורה ממוקדת.",
  "messages": [
    { "role": "user", "content": "={{ $json.body.question }}" }
  ]
}
מה זה {{ $json.body.question }}?

זה n8n expression — דרך להזריק data מ-node קודם. $json = הנתונים שזרמו ל-node הנוכחי. .body.question = השדה question בתוך body של ה-webhook. כל מה שמופיע ב-{{ }} מחושב דינמית.

3.4 Node 3 — Gmail עם התשובה של Claude

הוסף עוד Gmail Send אחרי ה-HTTP Request:

  • To: המייל שלך
  • Subject: תשובת Claude ל: {{ $('Webhook').first().json.body.question }}
  • Email Type: Text
  • Message: {{ $json.content[0].text }}

$json.content[0].text = חילוץ התשובה מהמבנה של Claude. זה בדיוק אותו ניווט שעשינו ב-Python עם response.content[0].text.

3.5 בדיקה עם curl

הפעל את ה-workflow ב-Listen for Test Event (כפתור בתחתית). ה-Test URL פעיל.

ב-Terminal של המאק:

Terminal
curl -X POST 'https://shai.app.n8n.cloud/webhook-test/ai-assistant' \
  -H 'Content-Type: application/json' \
  -d '{"question": "מה ההבדל בין SEO ל-SEM?"}'

(החלף את ה-URL ב-Test URL שלך מ-3.2.) תוך מספר שניות:

  1. n8n יראה את ה-nodes הופכים לירוקים
  2. תקבל מייל עם תשובת Claude

3.6 הפעלה לעבודה אמיתית

לחץ Active בפינה הימנית עליונה. עכשיו ה-Production URL פעיל 24/7. כל קריאה אליו — workflow רץ.

מה הרגע עשית

בנית שירות AI שלך שזמין דרך URL. אתה יכול לקרוא לו מ-Slack (פוסט אוטומטי), מהדפדפן, מ-bookmarklet, או אפילו מ-Python (requests.post). זה צעד אמיתי לקראת בניית AI agents.

Workflow 3 — בנה משהו שלך

45 דקות
הזמן ליצירה

עד עכשיו בנינו workflows יחד. עכשיו — אתה לבד. בחר אחד מהרעיונות למטה, או חשוב על משהו שיהיה שימושי לך באמת.

4.1 רעיונות לבחירה

אופציה A — Daily Tip Email

Workflow שרץ כל בוקר (Schedule Trigger), שולח prompt ל-Claude ("תן לי טיפ SEO ליום הזה"), ושולח את התשובה למייל. דרישה: לבחור שעה ב-Schedule node, להגדיר את ה-prompt, לחבר Gmail.

אופציה B — Wikipedia Researcher

אותו topic_researcher משיעור 3 — אבל ב-n8n. Webhook מקבל נושא → HTTP Request ל-Wikipedia → HTTP Request ל-Claude → מייל עם התוצאה. אתגר אמיתי — שילוב של 2 APIs ב-workflow אחד.

אופציה C — RSS to Email Digest

Schedule (פעם ביום) → RSS Read node (בלוג שאתה עוקב אחריו) → Claude לסיכום קצר → מייל עם 3 הסיכומים החדשים. יותר מורכב, אבל ערך מובהק.

אופציה D — משלך

חשוב על משהו שיהיה שימושי לעבודה שלך. הנוסחה: טריגר + 1-2 צעדי עיבוד + מוצא. אם זה לוקח לך פחות מ-45 דקות לבנות — מצוין. אם אתה תקוע — תפנה ל-Claude בצ'אט עם הרעיון, הוא יתן לך הצעות לארכיטקטורה.

4.2 בנה. תספר.

בחר אופציה, בנה את ה-workflow, הפעל לפחות פעם אחת בהצלחה. עדכן את ה-PROGRESS.md עם:

  • איזה workflow בנית
  • כמה זמן זה לקח
  • איפה היה הקושי הכי גדול
  • מה היה הרגע הכי מתגמל

5.2 Self-rating — דרג את עצמך 1-5

  • קריאת קוד Python — אני יכול לקבל סקריפט של 50 שורות שClaude כתב ולהבין מה הוא עושה.

  • חיבור ל-Claude API — אני יודע לכתוב prompt טוב, להריץ, ולעבד את התשובה.
