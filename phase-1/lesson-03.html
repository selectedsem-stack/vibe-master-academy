HTTP & APIs — איך תוכנות מדברות זו עם זו

30 דקות
המודל הבסיסי

כל פעם שאתה גולש לאתר, מצד שני יושב שרת. הדפדפן שולח request, השרת מחזיר response. זה HTTP. API זה אותו דבר רק שבמקום HTML יוצא JSON, ובמקום אדם — תוכנה שולחת את הבקשה.

1.1 מבנה של בקשת HTTP

חלקמה זהדוגמה
Methodסוג הפעולהGET, POST, PUT, DELETE
URLהכתובתhttps://api.example.com/users/42
Headersמטא־מידע על הבקשהAuthorization: Bearer xxx
Bodyהנתונים (רק ל-POST/PUT){"name": "Shai"}

1.2 GET vs POST — שני שימושים שונים

  • GET — "תן לי משהו". לקריאה בלבד. הפרמטרים בתוך ה-URL (?key=value).
  • POST — "תקבל ממני משהו וטפל בזה". יוצר/משנה. הנתונים ב-body.

כשאתה קורא ל-Claude, אתה שולח POST (מעביר prompt לטיפול). כשאתה מבקש תקציר מ-Wikipedia, אתה שולח GET (רק קורא).

1.3 Status codes — המספר שמגדיר אם זה הצליח

כל תשובת HTTP מתחילה במספר תלת־ספרתי. הקטגוריות החשובות:

טווחמה זה אומרדוגמאות
2xx✅ הצלחה200 OK, 201 Created
4xx❌ הבקשה שלך לא טובה401 Unauthorized, 404 Not Found, 429 Rate Limit
5xx❌ השרת הזה דפוק500 Server Error, 503 Unavailable
העיקרון לעבודה

תמיד תבדוק response.status_code לפני שאתה משתמש בתשובה. אם זה לא 200 — משהו השתבש, ותרצה לדעת מה. זה אחד הדפוסים הכי חשובים בעבודה עם APIs.

topic = input("איזה נושא לחקור? (באנגלית, למשל: Python, Beatport, SEO): ")

encoded_topic = quote(topic)

response = requests.get(url, timeout=10)

print(f"\nHTTP Status: {response.status_code}")

if response.status_code == 200:
    data = response.json()
    print(f"\n📚 {data['title']}\n")
    print(data["extract"])
    print(f"\n🔗 Read more: {data['content_urls']['desktop']['page']}")
elif response.status_code == 404:
    print(f"❌ לא נמצא דף Wikipedia עבור '{topic}'.")
else:
    print(f"❌ שגיאה לא צפויה: HTTP {response.status_code}")

2.3 קוראים שורה־שורה

שורה 1-2: import requests — מביא את הספרייה. from urllib.parse import quote — מביא רק פונקציה אחת (quote) מתוך urllib.parse. ההבדל: import X מביא הכל; from X import Y מביא רק Y.

שורות 5-6 — בניית ה-URL:

URL building
encoded_topic = quote(topic)
url = f"https://en.wikipedia.org/api/rest_v1/page/summary/{encoded_topic}"

quote() מצפינה תווים מיוחדים ב-URL. למשל "machine learning" → machine%20learning. תמיד תשתמש בזה כשמכניסים קלט משתמש ל-URL — אחרת רווחים ותווים מיוחדים ישברו.

שורה 8 — הקריאה עצמה:

requests.get
response = requests.get(url, timeout=10)
  • requests.get(url) — שולח בקשת GET לכתובת ומחזיר אובייקט Response
  • timeout=10 — תוותר אם השרת לא ענה תוך 10 שניות. תמיד תשים timeout, אחרת הקוד יכול להיתקע לנצח

שורות 12-21 — עיבוד התשובה:

  • response.status_code — המספר התלת־ספרתי
  • response.json() — ממיר את גוף התשובה (JSON string) ל-Python dict. ניווט במבנה: data["title"], data["content_urls"]["desktop"]["page"]
  • if / elif / else — שלוש מצבי תשובה: 200 (הצלחה), 404 (לא נמצא), אחר (שגיאה)

from __future__ import annotations

import os
import requests
from urllib.parse import quote
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv("../lesson-01/.env")
claude = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])


def fetch_wiki_summary(topic: str) -> dict | None:
    """מביא תקציר מ-Wikipedia עבור נושא נתון."""
    encoded = quote(topic)
    url = f"https://en.wikipedia.org/api/rest_v1/page/summary/{encoded}"
    response = requests.get(url, timeout=10)

    if response.status_code != 200:
        return None

    data = response.json()
    return {
        "title": data["title"],
        "extract": data["extract"],
        "url": data["content_urls"]["desktop"]["page"],
    }


def suggest_related_topics(summary: str) -> str:
    """שולח תקציר ל-Claude ומקבל 3 נושאים קשורים מעמיקים."""
    response = claude.messages.create(
        model="claude-haiku-4-5",
        max_tokens=512,
        system="אתה חוקר סקרן וחכם. עונה בעברית.",
        messages=[{
            "role": "user",
            "content": f"""קראתי את התקציר הזה מ-Wikipedia:

{summary}

הצע 3 נושאים קשורים שכדאי לחקור הלאה.
מבנה: 1. [נושא] — [למה] (שורה לכל אחד)."""
        }]
    )
    return response.content[0].text


topic = input("איזה נושא לחקור? ")

print(f"\n📖 מביא תקציר מ-Wikipedia...")
wiki = fetch_wiki_summary(topic)
if wiki is None:
    print(f"❌ לא נמצא דף Wikipedia עבור '{topic}'.")
    exit()

print(f"\n📚 {wiki['title']}\n")
print(wiki["extract"])
print(f"\n🔗 {wiki['url']}")

print(f"\n🤖 שואל את Claude על נושאים קשורים...")
suggestions = suggest_related_topics(wiki["extract"])

print(f"\n💡 3 נושאים שכדאי לחקור הלאה:\n")
print(suggestions)

Error Handling — מה קורה כשהדברים נשברים

45 דקות
למה זה חשוב

הסקריפט בחלק 3 קורס אם משהו משתבש — אין אינטרנט, Wikipedia מחזיר 500, Claude עם rate limit. בעבודה אמיתית זה לא קביל. הסקריפט בחלק הזה מטפל בכל זה בעדינות. אותה פונקציה, אבל עם רשת בטיחות.

4.1 try / except — מבנה הבסיסי

תבנית
try:
    # קוד שעלול להיכשל
    response = requests.get(url)
    data = response.json()
except SomeError as e:
    # קוד שרץ רק אם הניסיון נכשל
    print(f"משהו השתבש: {e}")

הרעיון: אופן הקריאה הוא תקין, אבל יש סיכון. try = "תנסה לבצע את זה". except = "אם השתבש, תעשה את זה במקום לקרוס". המשתנה e מכיל את אובייקט השגיאה (אם תרצה לקרוא אותו).

4.2 הקוד המוגן — safe_researcher.py

def fetch_wiki_summary(topic: str, retries: int = 2) -> dict | None:
    encoded = quote(topic)
    url = f"https://en.wikipedia.org/api/rest_v1/page/summary/{encoded}"

    for attempt in range(retries + 1):
        try:
            response = requests.get(url, timeout=10)

            if response.status_code == 404:
                return None  # נושא לא קיים — אין טעם לנסות שוב

            response.raise_for_status()  # זורק HTTPError אם הסטטוס 4xx/5xx
            data = response.json()
            return {
                "title": data["title"],
                "extract": data["extract"],
                "url": data["content_urls"]["desktop"]["page"],
            }

        except requests.exceptions.ConnectionError:
            print(f"   ⚠️  בעיית רשת בניסיון {attempt + 1}. ממתין 2 שניות...")
            time.sleep(2)

        except requests.exceptions.Timeout:
            print(f"   ⚠️  Timeout בניסיון {attempt + 1}.")
            time.sleep(2)

        except requests.exceptions.HTTPError as e:
            print(f"   ❌ שגיאת HTTP מהשרת: {e}")
            return None

    print(f"   ❌ נכשלו {retries + 1} ניסיונות.")
    return None

4.3 קוראים — חמישה דברים שצריך לזהות

1. הפרמטר retries: int = 2 — ערך ברירת מחדל. אם לא מעבירים, יוחל 2. אפשר לקרוא fetch_wiki_summary("X") או fetch_wiki_summary("X", retries=5).

2. for attempt in range(retries + 1): — מנסה retries+1 פעמים: פעם רגילה + retries ניסיונות חוזרים. range(3) = 0, 1, 2 — שלוש פעמים.

3. response.raise_for_status() — קיצור־דרך נחמד. אם הסטטוס הוא 4xx או 5xx, זה זורק HTTPError אוטומטית. במקום לכתוב if response.status_code >= 400: בעצמך.

4. שלושה except שונים — כל אחד מטפל בסוג שגיאה אחר:

  • ConnectionError — אין אינטרנט / DNS לא מצא את השרת
  • Timeout — השרת לא ענה בזמן
  • HTTPError — השרת ענה, אבל עם 4xx/5xx

בכל מקרה מטפלים שונה — בשגיאות רשת מנסים שוב, בשגיאת HTTP זה לרוב באג קבוע ולא נסה שוב.

5. time.sleep(2) — ממתין 2 שניות לפני הניסיון הבא. תמיד חשוב — אחרת תפציץ את השרת ב-100 ניסיונות בשנייה.

5.4 כללי אצבע ל-commit messages

  • פועל בהווה: "Add X", "Fix Y", "Update Z" (לא "Added", לא "Fixed")
  • קצר: עד 70 תווים בשורה הראשונה
  • למה, לא מה: לא "שיניתי 5 שורות ב-script.py" — אלא "Refactor retry logic to handle rate limits"
  • קומיט אחד = שינוי לוגי אחד: אל תערבב 5 דברים שונים בקומיט אחד. קל יותר לחזור אחורה אם כל קומיט הוא יחידה הגיונית

שאלה 1

בקוד הבא, מה יודפס אם response.status_code הוא 200?

קטע קוד
response = requests.get("https://example.com/api/users/1")
if response.status_code == 200:
    user = response.json()
    print(user["name"])
else:
    print("שגיאה")
תשובה

השם של המשתמש (מהשדה name ב-JSON). הבקשה הצליחה, התשובה נפרסה ל-Python dict, ומדפיסים את ה-name. אם הסטטוס היה 404 או 500, היה מודפס "שגיאה".

שאלה 2

מה ההבדל בין GET ל-POST?

תשובה

GET = קריאה. "תן לי משהו". פרמטרים בתוך ה-URL. POST = פעולה שמשנה משהו או מעבירה data כבד. הנתונים נשלחים ב-body של הבקשה. Wikipedia API קוראים GET, Claude API קוראים POST.

שאלה 3

בקוד הבא — מה יקרה אם הרשת תיפול?

קטע קוד
try:
    response = requests.get(url, timeout=10)
    data = response.json()
except requests.exceptions.ConnectionError:
    print("אין אינטרנט")
    data = None
תשובה

הקריאה ל-requests.get תזרוק ConnectionError. הקוד יקפוץ ל-except, ידפיס "אין אינטרנט", ויציב data = None. הסקריפט לא יקרוס — הוא ימשיך לקוד שאחרי ה-try/except.

שאלה 4

מהי הפקודה ב-Git שמראה לך מה השתנה אבל עדיין לא נשמר ב-commit?

תשובה

git status מציגה רשימת קבצים שהשתנו. git diff מציגה את השינויים עצמם — שורה אחר שורה — בתוך כל קובץ. תרגיל יומי: לפני כל commit, הרץ git status ואז git diff לפני שאתה רץ git add.
